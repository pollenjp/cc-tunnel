# ADRディレクトリ構成変更

**日時**: 2026-04-29T00:11:53+09:00

## 経緯

プロジェクト開始時からADR（Architecture Decision Record）は `adr/` 直下にフラットなディレクトリとして配置されていた。各ディレクトリには `report.md` やログファイルが格納される構成だった。

### 旧構成

```
adr/
  2026-04-19T074606JST_cmd_cctunnel_db_schema_refactor_001/
    report.md
  2026-04-19T09:06:54+00:00_cmd_cctunnel_disconnect_resilience_001/
    report.md
  ...
```

**問題点**:
- タイムスタンプ形式が統一されていない（JST、+0000、+00:00 が混在）
- 1ディレクトリに1ファイルしかない場合がほとんどで、ディレクトリ階層が冗長
- ファイル数増加に伴い `adr/` 直下が肥大化し、月別の一覧性が悪い
- `cmd_cctunnel_` プレフィックスが冗長

## 決定事項

### 新構成

```
adr/
  2026-04/
    2026-04-19T07:46:06+09:00_01_db_schema_refactor_001.md
    2026-04-19T08:24:23+09:00_01_arch_analysis_001.md
    ...
```

### 命名規則

- **ディレクトリ**: `adr/YYYY-MM/` （月別）
- **ファイル名**: `<ISO-8601 JST timestamp>_<index>_<task_title>.md`
  - タイムスタンプは全て JST（+09:00）に統一
  - `<index>`: 同一コマンドから生成されたADRの通し番号（2桁0パディング）
  - `<task_title>`: `cmd_cctunnel_` プレフィックスを除去した簡潔な名称

### index の採番ルール

- 基本は `01`（1コマンド1ADR）
- 同一タスク名で複数ディレクトリが存在した場合（例: terraform_frontend_deploy_001 が7つ）、出現順に `01`〜`07` を付与

## 変換の詳細

### タイムスタンプ変換

| 旧形式 | 変換方法 |
|---------|----------|
| `T074606JST` | そのまま JST として `T07:46:06+09:00` に整形 |
| `T125214+0000` | UTC→JST (+9h) 変換 |
| `T09:06:54+00:00` | UTC→JST (+9h) 変換 |
| タイムスタンプなし (`cmd_cctunnel_multitext_persist_001`) | ファイルの mtime から JST 変換 |

### コンテンツ統合

- ディレクトリ内の `.md` ファイルを結合して1つの ADR ファイルに
- `.log` ファイルがある場合は Appendix として末尾にコードブロックで追記
- 同一ディレクトリに複数レポート（例: 異なる ashigaru のレポート）がある場合も1ファイルに統合

## 結果

- 旧: 67ディレクトリ（`adr/` 直下にフラット配置）
- 新: 67ファイル（`adr/2026-04/` に集約）
- 全て JST タイムスタンプに統一
- `cmd_cctunnel_` プレフィックス除去済み
