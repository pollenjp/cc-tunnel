# ADR: POST /conversations/{id}/messages 401 修正

作成: ashigaru6, 2026-04-30
task: subtask_p401_002b

---

## 1. 401 エラーの根本原因

**症状**: 認証完了後、`GET /api/conversations` は 200、`POST /api/conversations/{id}/messages` は 401 になる。

**根本原因**: フロントエンド `apps/frontend/src/api/client.ts` の `sendMessage()` が生 `fetch()` を使い、`Authorization: Bearer <token>` ヘッダを付与していなかった。Go バックエンド `apps/cc-tunnel/internal/api/handler.go` の `Server.SendMessage` は Bearer トークンを必須とするため 401 を返す。

加えて、`openapi-fetch` クライアント経由の会話系 API (`listConversations`, `createConversation`, `getConversation`, `deleteConversation`) も Authorization ヘッダを送っていなかった（クライアントに middleware が未設定）。

## 2. GET が成功していた理由（セキュリティギャップ）

`Server.ListConversations`（および `CreateConversation`, `GetConversation`, `DeleteConversation`）には Bearer 認証チェックが実装されていなかった。そのため、Authorization ヘッダなしで GET /api/conversations が 200 を返していた。これは 401 バグではなく、別途認証漏れというセキュリティギャップであった。

## 4. 認証フロー全体像

```
[ユーザー]
  ↓ POST /app-auth/login (username)
[AppAuthLogin handler]
  → 32byte ランダム hex トークン生成
  → in-memory AppSession マップに保存
  → JSON レスポンスでトークン返却
[フロントエンド]
  → sessionStorage['app_auth_token'] に保存
  → client.ts middleware が全リクエストに Authorization: Bearer <token> を自動付与
[バックエンド ハンドラ]
  → bearerToken(r) でヘッダ抽出
  → session.get(token) でセッション確認
  → 未認証なら 401 返却
  → 認証済みなら処理続行
```

## 5. Option A / Option B 選定理由

### Option A: openapi-fetch middleware + 生 fetch 個別対応（採用）

`client.ts` に `getAuthToken()` ヘルパと `client.use({ onRequest })` を 1 つ追加し、`sendMessage` の生 fetch にもヘッダを足す。

**採用理由**:
- 1 ファイルの変更で `listConversations` / `createConversation` / `getConversation` / `deleteConversation` も同時に修正される。
- 既存呼び出し側（ChatView, Sidebar, App など）に手を入れない。
- `app-auth.ts` / `credentials.ts` の既存パターンと矛盾せず共存。

### Option B: 引数明示渡し（不採用）

`sendMessage(token, conversationId, content)` のように呼び出し側から `useAppAuth().token` を渡す方式。

**不採用理由**: 呼び出し側全部に変更が必要で変更範囲が大きい。React のレンダ依存も発生する。

## 6. 実施順序の重要性

`subtask_p401_002a`（フロントエンド Bearer header 注入）を先に完了させてからバックエンドの認証チェックを追加した。逆順（バックエンド先）だと、フロントエンドが Authorization ヘッダを送る前にバックエンドが認証必須になり、全エンドポイントが一時的に 401 に落ちる危険があった。
