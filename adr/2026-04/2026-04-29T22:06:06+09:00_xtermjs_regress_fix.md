# ADR: xterm.js 品質劣化修復 — AgentSettingsPage legacy auth 撤去

**日時**: 2026-04-29T22:06:06+09:00
**ステータス**: 採用

---

## 背景・根因

### 根因

`AgentSettingsPage` が `useAuth()` + `<AuthTerminal />` を引数なしで使う legacy 実装のまま放置されており、`subtask_ccauth_001b` の per-session routing 化以降は機能しない。

- `useAuth(conversationId = '')` — デフォルト空文字列
- `<AuthTerminal conversationId="" />` — 空 conversationId で `/api/auth/output?conversationId=&since=0` を叩く
- backend の `GetAuthOutput` は `getSessionClientOrRespond(w, r, h.executionProvider, '')` → `ErrSessionNotFound` → 404
- xterm Terminal は何も書き込まれず空表示のまま
- 250ms ごとに 404 が返り続ける

### `subtask_ccauth_001b` との関係

- `subtask_ccauth_001b` で per-session routing 化、共有 remote container 廃止
- `AuthGuard.tsx` は撤去対象に含まれたが `AgentSettingsPage` はスコープ外で legacy のまま残存
- `useAuth.ts` も同様に残存

### 参照 commit

殿が言及した commit `575ee04` は `git log` / `git reflog` / `git stash` に存在しない。
別環境または記憶違いと判断。当該 commit によるロールバックは不可能。

---

## 採用案: (A) AgentSettingsPage の auth 経路を撤去 + CredentialsLoginPage 誘導に統合

### 変更内容

| ファイル | 変更内容 |
|---------|---------|
| `apps/frontend/src/pages/AgentSettingsPage.tsx` | `useAuth` / `<AuthTerminal />` を撤去。`getCredentialsStatus` ベースの状態表示 + `createConversation` → navigate `/login/credentials?reason=missing&conversationId=<conv>` のボタンに変更 |
| `apps/frontend/src/pages/AgentSettingsPage.test.tsx` | useAuth/AuthTerminal mock を撤去。credential ベースの navigation テスト追加 |
| `apps/frontend/src/pages/CredentialsLoginPage.tsx` | タイトル「Claude 認証中...」+ reasonLabel サブテキスト化 + 「キャンセル」ボタン追加 |

### 採用理由

- `CredentialsLoginPage` は実 `conversationId` あり → `AuthTerminal` が正常な `/auth/output` ルートを使える
- `getCredentialsStatus` は `CredentialGuard` でも使用済みの安定した API
- frontend 局所改修のみ、backend 変更不要
- ログアウトボタンは本 cmd スコープ外として削除（将来 `DELETE /credentials` API ボタンとして別 cmd で再設計）

---

## 不採用案

### (B) AuthTerminal に empty conversationId フォールバック

**却下**: 共有 remote container が `subtask_ccauth_001b` で物理的に削除済み。空 conversationId で routing する宛先がもはや存在しない。フォールバック実装してもエラー隠蔽にしかならない。

### (C) cc-tunnel routing で empty conversationId 特殊ハンドリング

**却下**: per-session 化したものを部分的にロールバックすることは設計の一貫性を破壊する。backend の routing を特殊ケースで汚染するのは中長期でメンテナンスコストを増大させる。

### (D) commit 575ee04 のロールバック

**却下**: 当該 commit が git log/reflog/stash に存在しない。復元不能。共有 remote container 構成は `subtask_ccauth_001b/001c` で能動的に廃止されており、ロールバックは多数の subtask の取り消しを伴う。

---

## 結果

- `AgentSettingsPage` が `<AuthTerminal />` を埋め込まない（grep でゼロヒット）
- `AgentSettingsPage` が `useAuth` を import しない
- credential 未登録時「Claude Code でログイン」ボタン → `/login/credentials?reason=missing&conversationId=<conv>` に navigate
- `CredentialsLoginPage` は引き続き正常に xterm でカラー描画する（regression なし）
- 一時ガイド `docs/_xtermjs_regress_fix_guide.md` 削除済み
